# HEC Dictionary

## 1. 文档目标

本文档用于承载 `commerce-video-diagnosis` 中 HEC 体系的完整业务字典定义，作为：
- README 的外链主文档；
- `docs/input_output_schema.md` 的字典引用源；
- machine-readable JSON Schema 的语义依据；
- 业务、产品、研发、QA 的统一标签口径。

## 2. HEC 体系说明

HEC 是视频诊断输出中的核心标签体系，拆为三段：

- **H = Hook**：视频如何抓住用户注意力；
- **E = Effect**：视频主要通过什么方式完成说服；
- **C = CTA**：视频如何完成收口与转化推动。

在 `commerce-video-diagnosis` 中：
- `diagnosis.primary_hec` 使用 H / E / C 三段式主标签；
- `diagnosis.secondary_effects` 使用副 Effect 标签；
- `blueprint.primary_hec` 使用结构化标签代码；
- `diagnosis` 层默认同时输出 `label code + label_name`。

例如：

```json
{
  "primary_hec": {
    "hook_label": "H5",
    "hook_label_name": "反常识与悬念",
    "effect_label": "E5",
    "effect_label_name": "保姆级教程",
    "cta_label": "C1",
    "cta_label_name": "利益/价格逼单"
  }
}
```

## 3. 字典字段规范

每个 HEC 标签统一使用以下字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | string | 标签代码，如 `H1` / `E3` / `C5`。 |
| `name` | string | 标签中文名。 |
| `bucket` | string | 所属段，取值 `H` / `E` / `C`。 |
| `definition` | string | 标签定义。 |
| `appeal` | string | 主要诉求机制。 |
| `notes` | string | 使用边界或补充说明。 |

## 4. Hook Dictionary

### 4.1 Hook 概览表

| Code | 中文名 | 诉求机制 | 一句话定义 |
|---|---|---|---|
| `H1` | 痛点/焦虑直击 | 诉诸避害 | 直接暴露症状、制造麻烦与焦虑。 |
| `H2` | 利益/价格前置 | 诉诸趋利 | 直接抛出价格数字、极高性价比或福利开场。 |
| `H3` | 反差结果前置 | 诉诸视觉终效冲击 | 极具视觉落差的 Before & After 蜕变。 |
| `H4` | 即时操作展示 | 诉诸视觉过程冲击 | 动作即看点，开局直接一镜到底顺滑操作，无情绪铺垫。 |
| `H5` | 反常识与悬念 | 诉诸认知冲突与信息差 | 包括行为猎奇/反常规、认知/观点冲突、质疑打假/亲测验证。 |
| `H6` | 场景/人群代入 | 诉诸社交归属 | 包含剧情关系、节点时空、人群特征、热点话题代入。 |
| `H7` | 明星/权威同款 | 诉诸信任转移 | 借势明星、大 V 的脸或权威身份留人，包含同款妆造等。 |

### 4.2 Hook 单标签说明

#### `H1` 痛点/焦虑直击
- **定义**：直接暴露症状、麻烦、损失或焦虑点，先把“不解决会更糟”的感受抛给用户。
- **诉求机制**：诉诸避害。
- **备注**：重点在先打出问题压力，而不是先展示产品利益。

#### `H2` 利益/价格前置
- **定义**：开头直接抛出低价、福利、补贴、折扣或极高性价比信息。
- **诉求机制**：诉诸趋利。
- **备注**：核心是“先讲便宜或值”，不是结尾再补价格。

#### `H3` 反差结果前置
- **定义**：开头先给强烈的结果反差或 Before & After 终效冲击，吸引用户继续看。
- **诉求机制**：诉诸视觉终效冲击。
- **备注**：重点是结果落差，而不是操作过程本身。

#### `H4` 即时操作展示
- **定义**：开局直接进入动作展示，让操作过程本身成为看点。
- **诉求机制**：诉诸视觉过程冲击。
- **备注**：强调“立刻上手、直接开干”，通常没有情绪铺垫。

#### `H5` 反常识与悬念
- **定义**：通过行为猎奇、反常规观点、认知冲突、质疑打假或亲测验证等方式制造信息差。
- **诉求机制**：诉诸认知冲突与信息差。
- **备注**：重点是“这事跟你想的不一样”，不等于普通提问开场。

#### `H6` 场景/人群代入
- **定义**：通过剧情关系、时间节点、生活场景、人群身份或热点话题，让用户快速代入自己。
- **诉求机制**：诉诸社交归属。
- **备注**：重点是“这是在说我/说我的场景”，而不是单纯介绍人群标签。

#### `H7` 明星/权威同款
- **定义**：借助明星、大 V、专业人士或权威身份，快速建立注意力与信任。
- **诉求机制**：诉诸信任转移。
- **备注**：包含同款妆造、权威背书露脸、名人带入等形式。

## 5. Effect Dictionary

### 5.1 Effect 概览表

| Code | 中文名 | 诉求机制 | 一句话定义 |
|---|---|---|---|
| `E0` | 单点演示 | 诉诸低门槛感知与基础可见性 | 无门槛的普通状态、外观展示或常规使用步骤展示。 |
| `E1` | 效果测评 | 诉诸结果验证与可理解证据 | 有明确验证目标的常规实测验证。 |
| `E2` | 暴力实测 | 诉诸极限冲击与强证据压制 | 高压、破坏性、反常规的极端物理测试。 |
| `E3` | 对比/拉踩 | 诉诸相对优势与替代动机 | 跟竞品、其他通用方案、旧方案或过去的自己进行明确优劣比较。 |
| `E4` | 感官实证 | 诉诸感官代偿与临场体验 | 通过极致单感官输入触发大脑“通感”代偿，实现“以感代证”。 |
| `E5` | 保姆级教程 | 诉诸理解获得与操作掌控感 | 提供新方法、新认知或纠正旧方法的教程型说明。 |
| `E6` | 成分/参数科普 | 诉诸理性解释与专业可信度 | 通过成分、参数、机理或专有名词完成理性论证。 |
| `E7` | 产地溯源/工厂实录 | 诉诸源头透明与过程可信 | 通过原产地、种植/养殖基地、工厂流水线等源头事实建立信任。 |

### 5.2 Effect 单标签说明

#### `E0` 单点演示
- **定义**：展示普通状态、外观或常规使用动作，不承担强验证、强对比或强教育任务。
- **诉求机制**：诉诸低门槛感知与基础可见性。
- **备注**：属于低门槛展示，更多是“给你看一下”。

#### `E1` 效果测评
- **定义**：围绕明确验证目标，进行常规、可理解的效果验证或实测。
- **诉求机制**：诉诸结果验证与可理解证据。
- **备注**：重点是“验证是否有效”，但不走极端破坏测试。

#### `E2` 暴力实测
- **定义**：通过高压、破坏性、反常规或极端物理条件测试产品表现。
- **诉求机制**：诉诸极限冲击与强证据压制。
- **备注**：强调极限强度与冲击感，风险与边界也更高。

#### `E3` 对比/拉踩
- **定义**：把当前产品与竞品、旧方案、通用方案或过去状态进行明确优劣比较。
- **诉求机制**：诉诸相对优势与替代动机。
- **备注**：重点是“比出来更好/更差”，包含替代逻辑与拉踩风险。

#### `E4` 感官实证
- **定义**：通过极致的声音、触感、视觉细节等单感官刺激，让用户形成“像是自己感受到”的通感代偿。
- **诉求机制**：诉诸感官代偿与临场体验。
- **备注**：重点是“以感代证”，不是传统参数论证。

#### `E5` 保姆级教程
- **定义**：用步骤化说明教用户怎么做，或纠正旧方法、建立新认知。
- **诉求机制**：诉诸理解获得与操作掌控感。
- **备注**：重点是“教会你”，不是只展示结果。

#### `E6` 成分/参数科普
- **定义**：通过成分、参数、机理、专有名词或技术知识进行理性说明。
- **诉求机制**：诉诸理性解释与专业可信度。
- **备注**：重点是“为什么有效”，不是“看起来有效”。

#### `E7` 产地溯源/工厂实录
- **定义**：通过原产地、种植/养殖基地、生产工厂或流水线等源头事实建立可信度。
- **诉求机制**：诉诸源头透明与过程可信。
- **备注**：重点是“来源可信、过程真实”。

## 6. CTA Dictionary

### 6.1 CTA 概览表

| Code | 中文名 | 诉求机制 | 一句话定义 |
|---|---|---|---|
| `C1` | 利益/价格逼单 | 诉诸贪婪与稀缺 | 用绝对低价或强烈的价格落差进行逼单。 |
| `C2` | 福利/保障机制 | 诉诸安全感与风险厌恶 | 提供超预期价值或售后兜底。 |
| `C3` | 指令行动 | 诉诸行为捷径 | 下达极简的物理操作指令。 |
| `C4` | 人群/场景总结 | 诉诸身份认同与逻辑闭环 | 在尾部再次圈定人群或情境呼应。 |
| `C5` | 效果留白/情绪定格 | 诉诸感官余温与审美向往 | 用冲击力画面或治愈声音自然定格。 |

### 6.2 CTA 单标签说明

#### `C1` 利益/价格逼单
- **定义**：通过绝对低价、强价格落差、限时优惠等利益点推动立刻下单。
- **诉求机制**：诉诸贪婪与稀缺。
- **备注**：核心是强价格驱动，不只是普通提醒购买。

#### `C2` 福利/保障机制
- **定义**：通过赠品、补贴、额外权益、售后承诺或兜底机制降低决策风险。
- **诉求机制**：诉诸安全感与风险厌恶。
- **备注**：核心是“买了也不怕亏”。

#### `C3` 指令行动
- **定义**：结尾直接给出简短明确的操作指令，引导用户马上执行下一步动作。
- **诉求机制**：诉诸行为捷径。
- **备注**：重点是降低行动成本，例如点、拍、领、下单等。

#### `C4` 人群/场景总结
- **定义**：在结尾再次归纳适用人群或使用场景，形成逻辑闭环。
- **诉求机制**：诉诸身份认同与逻辑闭环。
- **备注**：重点是帮助用户完成“这就是给我看的”确认。

#### `C5` 效果留白/情绪定格
- **定义**：不强行逼单，而是用有余味的画面、声音或情绪氛围自然收尾。
- **诉求机制**：诉诸感官余温与审美向往。
- **备注**：适合以审美感、氛围感、治愈感完成最后定格。

## 7. 在 Schema / README / JSON Schema 中的承载方式

### 7.1 README
README 只放 HEC 体系摘要，不承载完整字典。

建议：
- 简述 H / E / C 含义；
- 放完整标签概览表；
- 链接到本文档。

### 7.2 `docs/input_output_schema.md`
Schema 文档中保留：
- 精简枚举表；
- `label + label_name` 的字段结构说明；
- 指向本文档的链接。

### 7.3 machine-readable JSON Schema
后续可在 JSON Schema 中增加：
- `enum`；
- `description`；
- 或 `oneOf + const + title`。

但 JSON Schema 只承担程序校验与工具消费，不替代本文档。
